Hosting Pelican site on Github: Best practices

  • Alexis Metaireau
  • Conan Doyle
  • Sun 03 October 2010
  • Mon 04 October 2010
  • Leave a comment

Starting out

  1. Create a new git repository at Github: https://github.com/new

    • Add Python to .gitignore

  2. Clone the newly created repository:

    $ git clone <repo_https_clone_url> <site_root>

  3. Create virtualenv (command only works if you have virtualenvwrapper):

    $ mkvirtualenv <venv_name>

  4. Install pelican (if you're not confident with reStructuredText yet, then install Markdown as well):

    $ pip install pelican markdown

  5. Navigate to <site_root> and execute the pelican-quickstart script:

    $ cd <site_root>
$ pelican-quickstart

    and answer the prompts according to your needs. Presumably, you want to upload your website using GitHub Pages, else you wouldn't be here. Make sure you answer y to generating a Fabfile/Makefile and to using Github pages.

Source control

If you're already familiar with Github Pages, you will will already know that content from the master branch will be used to build and publish your GitHub Pages site. Read this article if you have no idea what I'm talking about.

Approach #1: The Octopress Approach

  1. git checkout -b source

  2. git add Makefile develop_server.sh fabfile.py pelicanconf.py publishconf.py

  3. commit these files

  4. git push origin source

  5. create an article <root>/keyboard-review.md:

    Title: My First Review
Date: 2010-12-03 10:20
Category: Review

Following is a review of my favorite mechanical keyboard.
